# Developer Documentation

This documents describes the technical architecture, setup, and management of the Inception project for develope

## 1. Environement Setup

### Prerequisites
Ensure the folowing tools are installed on your host machine:
- **Docker**: container runtime
- **Docker compose**: Orchestration tool (usually included with the Docker installation)

To install Docker on Ubuntu, run the following commands ([*Others OS*](https://docs.docker.com/compose/install/linux/#install-using-the-repository)):
```sh
# Uninstall older versions
sudo apt remove $(dpkg --get-selections docker.io docker-compose \
docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)

# Add Docker's official GPG key:
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update

# Install the docker compose plugin
sudo apt-get update && sudo apt-get install docker-compose-plugin

# Verify the docker compose installation
docker compose version
```

### Configuration Files 

The project requires two types of configuration: environement variables and secrets

1. **Environment Variables**:
    Copy the example file and edit it to match your configuration (domain name, database name, usernames)
    ```sh
    cp srcs/.env.example srcs/.env
    nano srcs/.env
    ```

2. **Docker Secrets**:
    You must create a `secrets` directory at the root of the repository and populate it whith the necessary password files
    These files are read by docker compose to inject sensitive data into the containers
    ```sh
    mkdir secrets
    echo "YOUR_DB_PASSWORD" > secrets/db_password.txt
    echo "YOUR_DB_ROOT_PASSWORD" > secrets/db_root_password.txt
    echo "YOUR_WP_ADMIN_PASSWORD" > secrets/wp_admin_password.txt
    echo "YOUR_WP_USER_PASSWORD" > secrets/wp_user_password.txt
    echo "YOUR_FTP_PASSWORD" > secrets/ft_password.txt
    ```
    > **Note**: Replace the placeholder string with your actual passwords

## 2. Build and Launch

The project uses a `Makefile` to wrap `docker compose` commands for ease of use

-   **Build and Start (Detached)**:
    This commands pulls base images, builds custom images defined in `srcs/docker-compose.yml`, and starts the containers in the background
    ```sh
    make
    ```

-   **Build and Start (Foreground)**:
    Usefull for debugging, this runs the containers in the foreground so logs are visible immediately
    ```sh
    make debug
    ```

-   **Stop**:
    Stops the running containers
    ```sh
    make stop
    ```

## 3. Managing Containers and Volumes

### Relevant Commands

The `Makefile` provides several targets to manage the lifecycle of the application:

| Command | Description |
| :--- | :--- |
| `make ps` | Lists currently running containers |
| `make stop` | Stops the containers without removing them |
| `make clean` | Stops and removes the containers |
| `make fclean` | Stops, remove the containers, and **delete persistent data** (volumes) |
| `make re` | Executes `flcean` folowed by `make` (fresh start) |

### Debugging
To inspect the logs of the runing services:
```sh
docker compose -f srcs/docker-compose.yml logs -f
```
To access a specific container's shell:
```sh
docker exec -it <container_name> sh
```

## 4. Data Storage and Persistence

The project uses **Docker Volumes** to persist data generated by the containers
This ensures that database data and website files are not lost when a container is restarted or recreated

### Location
Volumes are managed by Docker and are stored by default in the `~/data/` folder

### Persistence Strategy
- **Database Volume**: Stores all MariaDB files
- **WordPress Volume**: Stores `wp-content` (uploads, plugins, themes)

> **Warning**: Running `make fclean` removes these volumes, resulting in complete data loss
